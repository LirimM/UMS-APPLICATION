# UMS-APPLICATION
University Management System created with Java Swing

**** PROJECT REQUIREMENTS ****

UNIVERSITY MANAGEMENT SYSTEM - UMS

Objective
A UMS where admin user can register one University, can change his
attributes and can delete the University.
A management user can add/view/change/delete students from University.

Users of the System
1. Admin
2. Management

Functional Requirements
1. Login / Logout
2. Create/Read/Update and delete (CRUD) University
3. Create/Read/Update and delete (CRUD) students
4. You should not be able to register the same student twice based on
personal no.
5. You should not be able to register more students then University
capacity
6. You should use SWING for building GUI
7. You should use Inheritance (parent class)
8. You should be able to search student based on personal no and name
9. UMS should contains method to find the oldest student in university
10. UMS should contains method to find student with the largest and
smallest name in university
11. Every functionality of UMS should be organized based on methods

Core Entities
1. User (Id, Name, Email, Gender, Address, Username, Role, CreateDate)
2. University (Id, Name, Address, Email, Tel, Students, Size, CreateDate)
3. Students (Id, Name, Address, Email, Tel, Gender, Place, Age,
CreateDate)
